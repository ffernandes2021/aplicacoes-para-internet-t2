ESPECIFICAÇÃO DE REQUISITOS (em construção)

Requisitos funcionais

[R1] Sessão: O acesso ao sistema deve ser realizado por meio das credenciais e-mail e senha, independente do tipo de usuário, iniciando sempre em uma nova sessão. Nesta o encerramento deve ser acessível ao usuário.

[R2] Manutenção de Senha: O sistema deve permitir a manutenção de senha, isto é, alteração e recuperação, independente do tipo de usuário.

[R3] Criação de cliente: O sistema deve permitir a criação de clientes por um usuário do tipo Cliente.

[R4] Manutenção de cliente: O sistema deve permitir a manutenção do cliente, isto é, a edição, por um usuário do tipo Cliente, além da ativação/inativação e exclusão por um usuário do tipo Gestor.

[R5] Criação de advogado: O sistema deve permitir a criação de advogados por um usuário do tipo Advogado.

[R6] Manutenção de advogado: O sistema deve permitir a manutenção de advogado, isto é, a pesquisa, por um usuário do tipo Cliente, a edição, por um usuário do tipo Advogado, e a ativação/inativação, além da exclusão, por um usuário do tipo Gestor.

[R7] Realização/Resposta de pergunta: O sistema deve permitir a realização de pergunta, por um usuário do tipo Cliente, além da possibilidade de responder por um usuário do tipo Advogado.

[R8] Manutenção de pergunta: O sistema deve permitir a manutenção de pergunta, isto é, a edição, a exclusão e a auditoria por um usuário do tipo Gestor.

[R9] Criação de sala: O sistema deve permitir a criação de sala por um usuário do tipo Advogado ou do tipo Gestor.

[R10] Acesso de sala: O sistema deve permitir o acesso à sala por um usuário do tipo Cliente ou do tipo Advogado.

[R11] Manutenção de sala: O sistema deve permitir a manutenção de sala, isto é, a edição e a exclusão por um usuário do tipo Gestor.

[R12] Publicação de artigo: O sistema deve permitir a publicação de artigo por um usuário do tipo Advogado.

[R13] Manutenção de artigo: O sistema deve permitir a manutenção de artigo, isto é, a pesquisa, por um usuário do tipo Advogado e exclusão e sinalização por um usuário do tipo Gestor.

[R14] Criação de plano: O sistema deve permitir a criação de plano por um usuário do tipo Gestor.

[R15] Manutenção de plano: O sistema deve permitir a manutenção de plano, isto é, a escolha por um usuário do tipo Advogado e a edição e a exclusão por um usuário do tipo Gestor.

[R16] Geração de extrato: O sistema deve permitir a geração de extrato por um usuário do tipo Cliente ou do tipo Advogado.

[R17] Criação de estagiário: O sistema deve permitir a criação de estagiário por um usuário do tipo Estagiário.
